### 5. Hardcoded Supabase Project ID in `next.config.ts`
**Severity: Low**  
```ts
hostname: 'nodkmtwchiivlwjktzis.supabase.co',
```
This is your actual Supabase project hostname hardcoded in `next.image` allowed domains. It's not a secret (it's a public hostname) but it means the config is environment-specific and will break if you ever migrate the Supabase project.

**Recommendation:** Use `process.env.NEXT_PUBLIC_SUPABASE_URL` to derive the hostname dynamically:
```ts
hostname: new URL(process.env.NEXT_PUBLIC_SUPABASE_URL ?? 'https://placeholder.supabase.co').hostname,
```

---

### 6. `/negotiate`, `/job-analysis`, `/resume-insights` Missing from Middleware `protectedRoutes`
**Severity: Low (mitigated)**  
These three routes are NOT in the `protectedRoutes` array in `lib/supabase/middleware.ts`. They are individually protected via server-side `redirect('/login')` in their `page.tsx` files, so there is no actual auth bypass — but the middleware is inconsistent and a future developer adding a new page under these paths could forget the server-side guard.

**Recommendation:** Add to `protectedRoutes` in middleware:
```ts
'/negotiate',
'/job-analysis',  
'/resume-insights',
```
